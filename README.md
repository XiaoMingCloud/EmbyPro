# EmbyPro

EmbyPro 是一个基于 Emby 官方开放 API 开发的 Android 第三方客户端，目标是提供接近官方付费客户端的完整体验，同时在播放器交互上做出更适合移动端沉浸式观看的增强。

项目当前已经完成了服务器接入、媒体库浏览、视频详情、播放器、封面缓存、动态取色 UI 等核心能力，并持续围绕“更顺手的连续观看体验”进行优化。

## 项目特点

- 基于 Emby 官方 API 实现服务器连接、登录、媒体库读取、视频详情、收藏、播放进度同步等能力
- 支持服务器列表管理，包括重新登录、编辑、修改头像、删除等操作
- 支持媒体库、继续观看、收藏、类型、标签、合集、文件夹等浏览方式
- 支持视频详情页动态主题取色，播放按钮会根据封面颜色自动变化
- 支持本地缓存媒体库列表、封面和视频基础信息，降低重复请求

## 核心亮点

### 1. 连续播放体验

相比很多市面上的 Emby 客户端更偏“点播式”观看，EmbyPro 更强调连续观看：

- 支持连播开关
- 当前视频播放结束后自动无缝切换到下一个视频
- 连播开启时会提前预缓存后续视频，减少切换等待
- 从播放器返回时，会自动回到“当前实际播放到的视频详情页”，而不是最初进入的那一条

### 2. 上下切换播放

这是项目里非常有辨识度的一项体验增强：

- 竖屏播放时支持像短视频一样上下切换视频
- 上滑下一个，下滑上一个
- 与连播能力结合后，可以形成非常顺滑的连续浏览和连续观看体验

这一点在传统 Emby 客户端里并不常见，也是 EmbyPro 希望做出的差异化方向。

## 已实现功能

- Emby 服务器连接与登录
- 用户头像拉取与头像修改
- 服务器首页与媒体库浏览
- 媒体库分页加载与排序
- 视频详情页与章节展示
- 收藏 / 取消收藏
- 播放进度读取与继续播放
- 自定义播放器手势
- 横竖屏播放与手动画面旋转
- 连播、上下切换播放、预缓存
- 本地封面与列表缓存
- Dynamic Color 风格的详情页动态配色

## 技术方向

- Android 原生 Kotlin 开发
- 基于 Emby REST API
- Media3 / ExoPlayer 播放器方案
- 本地缓存 + 会话持久化
- 面向移动端沉浸式交互设计

## 项目目标

EmbyPro 不只是一个“能看”的 Emby 客户端，而是想做一个更适合手机长时间观看、连续观看、沉浸式观看的 Android 客户端。

在保留 Emby 生态兼容性的同时，项目会继续围绕以下方向迭代：

- 更完整的播放器能力
- 更细腻的动态主题与界面动效
- 更顺滑的连续播放与切换体验
- 更适合移动端单手操作的交互设计
